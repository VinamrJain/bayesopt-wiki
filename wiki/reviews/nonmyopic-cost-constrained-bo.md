- [nonmyopic-cost-constrained-bo-1] nit (open) — The opening paragraph (line 20) calls the guarantee 'a clean optimality guarantee (rollout never hurts the base policy)'. It is a policy-improvement guarantee (no worse than the base policy), not an optimality guarantee for the CMDP — the note itself is precise about this later ('rollout never does worse than the base policy', Thm). The intro wording slightly overstates it. Change 'a clean optimality guarantee' to 'a clean improvement guarantee' (or 'policy-improvement guarantee') in the intro for consistency with the theorem statement.
- [nonmyopic-cost-constrained-bo-2] nit (open) — lee2021's actual motivation for going non-myopic is that naive EIpu is biased toward cheap points and fails badly when the optimum is expensive (background.tex: dividing by cost penalizes EIpu away from expensive optima; KNN histogram; cmdp.tex Fig 1 footnote that EIpu would evaluate exclusively in the cheap region). The note presents EIpu only positively, as the principled cheap-first component of the base policy, and never surfaces that lee2021's central empirical finding is that plain EIpu underperforms — the very reason rollout is needed. This failure mode is owned by [[ei-per-unit-cost]], so the omission is defensible, but a one-line nod would sharpen the 'why rollout over plain EIpu' logic. Optionally add one sentence (or a cross-link cue) noting lee2021 motivates the CMDP rollout by EIpu's cheap-point bias failing when optima are expensive, deferring the detail to [[ei-per-unit-cost]].
- [nonmyopic-cost-constrained-bo-3] nit (open) — lam2016 is in the sources: frontmatter but contributes no derivation to the body — it appears only as a one-line lineage attribution ('the MDP-for-BO framing from lam2016', line 183), with the actual MDP/rollout material sourced from lee2021 and routed to [[bo-as-dynamic-program]]. The attribution is accurate (lam2016 is the originator of MDP-for-BO and of rollout-for-BO), so listing it is defensible under 'material appears in the body', but it is a thin, pointer-only use rather than a derivation source. Acceptable as-is given the accurate lineage attribution; if the house rule on sources: is read strictly (derivation material only), demote lam2016 to an inline citation and drop it from sources:. No content change to the math.
