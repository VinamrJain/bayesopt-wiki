- [budget-constrained-dp-2] blocker (open) — The symbol B is defined incorrectly in the canonical notation.md and the note silently inherits the contradiction. notation.md (line 93) says B is the 'expected cost budget (random-cost setting): feasibility E[sum_n c(x_n)] <= B'. But astudillo2021 (§3, raw line 114; Prop 1 proof, supplement lines 113-124) uses B as a HARD, pathwise cap: sum_i c(x_i) <= B deterministically. The horizon N_B = sup{k: s(D_k) <= B} is a random stopping time precisely because the realized cumulative cost s(D_k) is random against a FIXED B -- not because B bounds an expectation. This note (delta and body, e.g. 's(D_n) <= B', N_B definition) correctly uses the hard pathwise cap, so the note and notation.md mathematically disagree about what B means, breaking the 'note + notation.md is self-contained' invariant. notation.md is the wrong one; there is no expected-budget / E[sum c]<=B formulation anywhere in astudillo2021. Fix notation.md line 93 to define B as a hard total-cost cap (feasibility sum_n c(x_n) <= B almost surely), distinguishing it from tau only by the random-cost modeling, not by an expectation. As a stopgap within this note, add one line to the Notation delta stating B is a hard pathwise budget (s(D_n) <= B) so the note does not silently contradict the inherited table.
- [budget-constrained-dp-3] nit (open) — The EIpu-failure paragraph (lines 138-143) says EI's numerator 'overvalu[es] the small, near-certain gains of low-variance points.' In astudillo's exact construction the low-variance points have a SMALL EI numerator (eps*E[Z^+]); the ratio ties/beats the expensive point only because the denominator cost eps is equally small. The decisive mechanism is the denominator, not numerator overvaluation. The note does state the denominator amplification, but the 'numerator overvalues' framing (lifted from astudillo's looser prose) can mislead a reader doing the algebra, since EI(cheap)<EI(expensive) here. Optionally tighten: note that for low-variance points the EI numerator is itself small (~eps*E[Z^+]); it is the equally small cost in the denominator that lifts the ratio to E[Z^+], so EIpu cannot distinguish a near-certain tiny gain from a genuine one. This makes the 'cost in the denominator' point exact rather than relying on 'overvaluation.'
